17] how to set wget proxy
    if you want to install any app through the terminal and internet is through proxy then you have to set wget proxy 
    This is valid if there is some wget word inside the command. 
    usually following error comes without setting wget proxy 
            407 proxy authentication is required.

    
You need to edit the wgetrc file located at /etc/wgetrc.

Use your favourite text editor, ie: nano /etc/wgetrc to edit it

Look for the http_proxy tag and remove the # in front of it, add the proxy server after the = sign, ie:

http_proxy=http://foo.proxy:8080

Then search for the use_proxy tag and remove the # in front of it, save the file.

wget will be then using the specified proxy server.

If you can't change system configuration with your user you can copy the current configuration to a place you can edit it, ie: cp /etc/wgetrc ~ and use the --config=~/wgetrc option to force wget to use the configuration you altered instead of the global configuration.

 62] Files having ".a" extension
  
     https://en.wikipedia.org/wiki/Static_library

     ".a" files are static libraries typically generated by the archive tool (ar utility in linux). They are statically linked i.e. when you are
     using -c option while compiling with gcc. 

     Static library also means whenever you change the library, you need to compile and build the code once again.

     when you are using some static library code, it is taken from the library and is embeded into your binary. But when you use dynamic libraries i.e.
     .so files, some library funciton code is not embeded into your binary file, instead it is just referenced and loded/added to your code at
     runtime. They are linked dynamically means they are referenced after creation of .o files.   

     while compiling, we usually include the header files associated with that static library and link to the library.

     Static library is usually collection of ".o" files.

     To use static libraries with gcc, collect all .a files in a lib/ folder and then link with "-Llib/" (note there is no white space between -L 
     and directory) and "-l<name of specific library>".

     Collection of all .a files into lib/ is optional. Doing so makes for better looking directories with nice separation of code and libraries,
     
     We can see the contents of a "foo.a" static library by using "ar tv foo.a"
